# ORBISTIUM

ORBISTIUM is the canonical world-state repository for the Verifrax system.

## What this repository is

ORBISTIUM stores accepted state under law already defined elsewhere:
- current graph snapshot
- current epoch snapshot
- trust summary
- contradiction inventory
- repair inventory
- quarantine inventory
- proof bundle references
- immutable epoch history

## What this repository is not

ORBISTIUM does not define constitutional law.
ORBISTIUM does not execute reconciliation.
ORBISTIUM does not collect GitHub, npm, HTTP, or evidence telemetry.
ORBISTIUM does not bypass accepted epoch flow with ad hoc current-truth edits.

## Dependency direction

SYNTAGMARIUM -> ORBISTIUM <- CONSONORIUM

Law is loaded from SYNTAGMARIUM.
Accepted reconciliation outputs are written here by CONSONORIUM.

## Root rule

No active Verifrax surface may contradict the reconciled epoch.

## Status

This repository is the accepted state store.
It is not the law source.
It is not the runtime.

## Sovereign wiring

**Inputs**
- accepted reconciliation outputs from CONSONORIUM
- law context defined by SYNTAGMARIUM

**Outputs**
- canonical accepted current state
- immutable epoch history
- state references consumed by public proof and verification surfaces

## Acceptance boundary

ORBISTIUM stores accepted state only.

CONSONORIUM may propose candidates.
CONSONORIUM may not self-ratify accepted state.
Direct ad hoc edits to accepted current truth violate the sovereign boundary.

## Non-bypass rule

No runtime candidate, projection artifact, or operator shortcut may outrank accepted epoch state once written here.
